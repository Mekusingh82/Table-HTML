Web Browser

A web browser is a software application used to access, retrieve, and display information from the World Wide Web, including websites, images, and videos. It acts as an interface between the user and the web server, interpreting HTML code to render web pages. Popular examples include Google Chrome, Safari, Firefox, and Microsoft Edge.

How a Web Browser Works

When you type a URL like https://www.google.com and hit Enter, the browser goes through a high-speed process to deliver the page to your screen.

1. The DNS Lookup (Finding the Address)

The browser sends an HTTP/HTTPS Request to that server. Think of this as a formal letter saying: "Hello, I would like to see the home page files for this website, please."

The browser contacts a DNS (Domain Name System) server.

It asks: "What is the IP address for this name?" Once it gets the address, it knows exactly which server in the world to talk to.

2. The Request (Sending the Letter)

The browser sends an HTTP/HTTPS Request to that server. Think of this as a formal letter saying: "Hello, I would like to see the home page files for this website, please."

3. The Response (Receiving the Raw Material)

The server sends back a Response. This usually includes:

HTML

The structure (the text and skeleton).

css

The style (colors and fonts).

JavaScript:

The interactivity (buttons and animations).

4. Parsing & Rendering (Building the Page)

This is where the magic happens inside the Browser Engine. The browser reads the code and builds two internal maps:

DOM (Document Object Model): A tree structure of all the HTML elements. CSSOM (CSS Object Model): A map of how those elements should look.

5. The Final Paint

The browser combines the DOM and CSSOM to create a Render Tree. It calculates exactly where every pixel should go on your screen (Layout) and then literally "paints" the colors, text, and images onto your display.
